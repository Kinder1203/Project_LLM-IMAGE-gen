# AGENTS.md

이 파일은 이 저장소에서 Codex가 따라야 할 기본 작업 규칙입니다. 하위 폴더에 별도 `AGENTS.md`가 없으면 이 파일을 기준으로 작업합니다.

## 작업 우선순위
1. 가장 가까운 `README.md`와 `AGENTS.md`를 먼저 읽고 현재 폴더의 규칙을 확인합니다.
2. 파이프라인 계약이 필요하면 `src/llm_pipeline/README.md`를 단일 기준 문서로 봅니다.
3. 계약이 확정된 뒤 코드, 테스트, 데모 스크립트 순으로 수정합니다.
4. 하위 문서는 실제로 구조 설명이나 폴더 전용 규칙이 필요할 때만 추가합니다.

## 수정 원칙
- 루트 `README.md`는 사람용 온보딩 문서로 유지합니다.
- `src/llm_pipeline/README.md`는 파이프라인 흐름, 액션, 상태, 검수 정책의 단일 기준 문서입니다.
- 추론 백엔드는 `vLLM OpenAI-compatible endpoint`를 기준으로 유지합니다. 예전 전용 추론 어댑터를 다시 도입하지 않습니다.
- ComfyUI JSON 템플릿은 저장소에서 직접 수정하지 않습니다. 런타임 주입 방식 전제를 유지합니다.
- 같은 계약 내용을 여러 문서에 길게 중복하지 않습니다.
- 문서는 계약, 실행 방법, 의존성, 경로, 명령이 실제로 바뀐 경우에만 갱신합니다. 내부 구현만 바뀌고 외부 계약이 유지되면 문서를 불필요하게 수정하지 않습니다.
- 새 폴더 문서는 필요성이 명확할 때만 만듭니다.
- 실제 저장소에 없는 파일명이나 경로를 문서화하지 않습니다.
- 실제 파일명, 경로, 설정값, 환경변수 이름은 코드와 설정 파일에서 직접 확인한 뒤 사용합니다. 추측으로 새 이름을 만들지 않습니다.
- 현재 작업과 직접 관련 없는 파일은 건드리지 않습니다. 광범위한 정리나 리팩터링은 명시적 필요가 있을 때만 수행합니다.
- 검수 오류나 HTTP 오류를 배경 보정 같은 정상 흐름으로 표현하지 않습니다.

## 테스트 명령
기본 검증 명령:

```bash
python -m unittest tests.test_pipeline_contract
```

수동 데모 확인:

```bash
python test_run.py
```

RAG 규칙 재적재:

```bash
python -m src.llm_pipeline.scripts.db_feeder
```

임베딩 모델을 바꾼 뒤에는 위 명령으로 Chroma 인덱스를 다시 적재합니다.

- 수정한 범위와 직접 관련된 테스트나 검증을 우선 실행합니다. 검증을 생략했거나 실행하지 못했으면 이유를 명시합니다.

## 금지사항
- 문서 여러 곳에 동일 계약을 복제하지 않습니다.
- 가장 가까운 문서를 읽지 않고 상위 규칙만으로 단정하지 않습니다.
- 불필요한 폴더별 `AGENTS.md`를 남발하지 않습니다.
- 실제 파일명과 다른 템플릿 이름을 테스트나 문서에 쓰지 않습니다.
- 대규모 로그, JSON 템플릿, 생성 산출물을 목적 없이 전부 열지 않습니다.

## 판단 원칙
- 근거 없는 낙관적 해석이나 과도한 비관적 해석을 하지 않습니다.
- 사용자의 가정이나 기존 문서를 그대로 사실로 단정하지 않고, 코드·테스트·로그·설정값·공식 문서 등 검증 가능한 근거를 우선 확인합니다.
- 확실하지 않은 내용은 단정하지 않고, 확인된 사실과 추정 사항을 구분합니다.
- 결론은 아부나 과장 없이 핵심 근거와 함께 간결하게 제시합니다.

## 출력 형식
- 기본 응답은 한국어로 작성합니다.
- 구현 완료 보고는 가능하면 `변경 사항`, `검증`, `남은 리스크` 순으로 짧게 정리합니다.
- 코드 리뷰 요청에는 findings-first로 답합니다.
- 문서 작업이라도 검증을 했으면 실행한 명령을 명시합니다.

## 토큰 및 컨텍스트 최적화
- 작업 전 가장 가까운 `README.md`와 `AGENTS.md`부터 읽습니다.
- 전체 파일 열람보다 `rg`, 필요한 함수 범위 읽기, 특정 라인 범위 확인을 우선합니다.
- ComfyUI JSON 템플릿은 실제 수정이나 계약 확인이 필요할 때만 엽니다.
- 테스트 로그는 실패 구간 위주로만 읽고 전체를 재인용하지 않습니다.
- 단일 기준 문서 내용을 다른 응답에 장문으로 다시 복사하지 않습니다.
- 하위 폴더에 별도 규칙이 없으면 루트 `AGENTS.md`만 따릅니다.
